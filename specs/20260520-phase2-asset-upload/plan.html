Implementation Plan — P2 Asset Upload

Ordered steps to implement requirements R2.1–R2.12. Each step should leave the repo in a runnable state.

Branch: phase/2-asset-upload (off master) · merge via PR only · Git workflow
Agent skills: productizer-p2-asset-upload + productizer-git-workflow + productizer-phase-validate

Step 0 — Branch from master

git fetch origin
git checkout master && git pull origin master
git checkout phase/2-asset-upload 2>/dev/null || git checkout -b phase/2-asset-upload
git branch --show-current   # must print: phase/2-asset-upload

Include pnpm db:studio in package.json if not yet on branch (dev tooling).

Step 1 — Dependencies & schema

  1. pnpm add sharp (image processing; use default install for platform)
  2. Add designAssets table and referenceAssetId on designConcepts per R2.1–R2.2
  3. pnpm db:generate && pnpm db:migrate

Step 2 — Storage & processing layer

  1. src/lib/assets/storage-paths.ts — safe keys under assets/{id}/
  2. src/lib/assets/process-image.ts — generate thumb, card, full buffers via sharp
  3. src/lib/assets/repository.ts — insert asset row, link concept reference
  4. Extend src/lib/concepts/repository.ts to return referenceAssetId and join metadata if needed

Step 3 — Validation schema

  1. src/lib/schemas/asset.ts — allowed MIME types, max bytes, variant enum
  2. Vitest tests for schema and path helpers

Step 4 — REST API routes

  1. src/app/api/assets/route.ts — POST multipart (parse with request.formData())
  2. src/app/api/assets/[id]/route.ts — GET metadata
  3. src/app/api/assets/[id]/file/route.ts — GET file stream by variant query
  4. Consistent JSON errors { error: string }
  5. Verify with curl (see validation.html)

Step 5 — Client hook & component

  1. src/hooks/use-upload-asset.ts — FormData mutation + cache invalidation
  2. src/components/assets/asset-upload.tsx — file input, preview <img src=.../file?variant=card>
  3. Wire into src/app/(studio)/concepts/[id]/page.tsx

Step 6 — Studio polish

  1. Update studio home badge to P2 where appropriate
  2. Optional: thumb on concept-card.tsx (R2.S2)
  3. README: document upload flow and pnpm db:studio

Step 7 — Tests & validation script

  1. e2e/fixtures/sample.png — tiny PNG for Playwright
  2. e2e/asset-upload.spec.ts — upload flow with data-testid
  3. Finalize scripts/validate-p2.sh curl upload smoke
  4. ./scripts/validate-p2.sh exit 0 before PR
