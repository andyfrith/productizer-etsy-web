3. Agent — curl reference (debug)

# Upload (replace CONCEPT_ID and path to image)
curl -s -X POST http://localhost:3000/api/assets \
  -F "file=@e2e/fixtures/sample.png" \
  -F "conceptId=CONCEPT_ID" | jq .

# Metadata
curl -s http://localhost:3000/api/assets/ASSET_ID | jq .

# Serve card rendition
curl -s -o /dev/null -w "HTTP %{http_code}\n" \
  "http://localhost:3000/api/assets/ASSET_ID/file?variant=card"

# List references for a concept
curl -s "http://localhost:3000/api/concepts/CONCEPT_ID/references" | jq .

# Set preview image
curl -s -X PUT "http://localhost:3000/api/concepts/CONCEPT_ID/preview" \
  -H "Content-Type: application/json" \
  -d '{"assetId":"ASSET_ID"}' | jq .

# Remove one reference
curl -s -X DELETE "http://localhost:3000/api/assets/ASSET_ID?conceptId=CONCEPT_ID" | jq .

4. Playwright — upload automation (required)

File: e2e/asset-upload.spec.ts

  • Create concept via UI or API fixture
  • Open /concepts/:id — upload control visible (data-testid="asset-upload")
  • Set file input — pending preview (asset-preview-local, data-reference-kind="pending")
  • Remove pending — Undo toast restores asset-preview-local
  • Upload — saved preview; remove saved — Undo toast restores via PUT …/reference
  • Success toast or stable state after each action

5. Manual (≤ 3)

  • Docker Desktop running if script reports Postgres unhealthy
  • Port 3000 / 5432 not blocked by other processes
  • GitHub PR merge after review (agent does not merge)

6. PR checklist (paste in body)

  • ./scripts/validate-p2.sh — all PASS
  • Branch phase/2-asset-upload
  • Screenshot or note: upload + preview on concept detail
  • No secrets in diff
