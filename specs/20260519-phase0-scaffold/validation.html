Validation — P0 Scaffold

Use this page before opening a PR. Automated gate must pass in CI or locally; manual checks confirm UX and developer experience.

PR-ready when: Branch is phase/0-scaffold, automated gate passes, manual checks done, requirements traced in PR, no secrets in diff. Merge via PR to master only (git workflow).

Git branch

git branch --show-current    # → phase/0-scaffold
git log master..HEAD --oneline   # commits not yet on master
  • All work committed on phase/0-scaffold, not master
  • Branch was created from latest master
  • PR targets master (not another feature branch)

Environment setup (once per machine)

node -v          # v22.x
pnpm -v
docker compose version
cp .env.example .env.local
docker compose up -d
docker compose ps   # postgres healthy

Automated gate

Run from repo root after pnpm install:

pnpm db:migrate
pnpm lint
pnpm test
pnpm build
pnpm test:e2e
  • pnpm db:migrate — migrations apply cleanly against Compose Postgres
  • pnpm lint — zero errors
  • pnpm test — all Vitest tests pass
  • pnpm build — production build succeeds
  • pnpm test:e2e — Playwright smoke passes (dev server via webServer config)

Runtime smoke (manual commands)

pnpm dev
# separate terminal:
curl -s http://localhost:3000/api/health/db | jq .
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/
  • Studio home loads at http://localhost:3000
  • DB health endpoint returns ok: true (or documented error when DB down)
  • Stopping Postgres (docker compose stop) shows graceful error state on home (not a blank crash)
  • storage/ exists or is created; not tracked by git

Manual UX & design checks

  • Accent color reads as Etsy-adjacent orange on primary actions
  • Dark studio shell feels consistent with constitution / brainstorm aesthetic
  • Card layout on home is polished (spacing, typography, not default Next.js template)
  • Skeleton loading state visible during DB check (or brief demo)
  • Sonner toast can be triggered (dev-only button or documented test) — optional but recommended
  • Framer Motion entrance is subtle, not distracting
  • Keyboard focus visible on interactive elements
  • Page title and meta reflect “Productizer” branding
