Requirements — P0 Scaffold

Functional and technical requirements for bootstrapping Productizer. Satisfies roadmap phase 0 and unblocks P1 Concept CRUD.

Locked decisions
  • Phase: P0 — full roadmap scaffold scope
  • Runtime: pnpm, Node 22 LTS
  • Framework: latest stable Next.js App Router, TypeScript, --src-dir
  • Validation bar: automated scripts + manual UX/design checks
  • Spec folder: specs/20260519-phase0-scaffold/
  • Git branch: phase/0-scaffold → merge via PR to master

Goal

A new contributor can clone the repo, start Postgres with Docker, run migrations, and launch a polished minimal studio shell at localhost:3000 with working lint, unit tests, E2E smoke, and production build.

Prerequisites (already in repo)

  • docker-compose.yml — Postgres 16 with healthcheck and named volume
  • .env.example — DATABASE_URL, STORAGE_ROOT, AI provider placeholders
  • Constitution docs under specs/

Requirements (SHOULD)

  • .node-version or engines + packageManager field for Corepack/pnpm
  • Prettier or existing formatter aligned with Next.js ESLint
  • components/mini-mock/ placeholder component (static, for P6 dashboard reuse)
  • GitHub Actions workflow stub (lint + test) — optional if deferred, note in PR

Out of scope (P0)

  • Concept CRUD UI and design_concepts table (P1)
  • Asset upload API (P2)
  • Variation gallery, AI generation jobs, poster mock (P3–P5)
  • Auth, Etsy OAuth, niche research (P11+)
  • Production deploy / cloud storage
  • Live calls to Nano Banana or other image APIs

Non-functional

  • UX polish: Dark-friendly studio shell consistent with constitution mock aesthetic; loading and error states on DB health check
  • Security: No secrets committed; .env.local gitignored
  • Performance: pnpm build succeeds without critical warnings
  • Accessibility: shadcn defaults; focus visible on interactive elements

Dependencies

  • Docker Desktop (or compatible) for Postgres
  • Node 22, pnpm 9+
  • Ports: 3000 (Next), 5432 (Postgres)
