Implementation Plan — P0 Scaffold

Ordered steps to implement requirements R0.1–R0.15. Each step should leave the repo in a runnable state.

Branch: phase/0-scaffold (off master) · merge via PR only · Git workflow
Agent skill: Use productizer-p0-scaffold + productizer-git-workflow when implementing in Cursor.

Step 0 — Branch from master

git fetch origin
git checkout master && git pull origin master
git checkout phase/0-scaffold 2>/dev/null || git checkout -b phase/0-scaffold
git branch --show-current   # must print: phase/0-scaffold

All commits for P0 (specs + app scaffold) stay on this branch until the PR merges.

Step 1 — Toolchain & Next.js app

  1. Add .nvmrc with 22 and packageManager in package.json.
  2. Scaffold Next.js in repo root (existing files preserved): 
    pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"
# If prompted about overwriting, merge carefully — keep README, docker-compose, specs/
  3. Verify pnpm dev serves default page.

Step 2 — shadcn/ui & design tokens

  1. pnpm dlx shadcn@latest init — New York style, zinc/slate base, CSS variables.
  2. Map accent to Etsy-adjacent orange in globals.css / theme (--primary ≈ #f56400).
  3. Add components: button, card, skeleton, sonner.
  4. Install framer-motion; wrap studio hero in a simple motion.div fade-in.

Step 3 — TanStack Query & providers

  1. pnpm add @tanstack/react-query
  2. Create src/app/providers.tsx with QueryClientProvider.
  3. Wire providers in root layout.tsx; add <Toaster /> from Sonner.

Step 4 — Docker & env (verify)

  1. Confirm docker compose up -d → Postgres healthy.
  2. cp .env.example .env.local
  3. Ensure .gitignore includes .env.local, storage/, .next/, node_modules/.
  4. Ensure storage/ directory is created at runtime if missing.

Step 5 — Drizzle ORM

  1. pnpm add drizzle-orm postgres · pnpm add -D drizzle-kit
  2. src/lib/db/index.ts — client from DATABASE_URL
  3. drizzle.config.ts at repo root
  4. Initial schema: minimal table (e.g. app_meta key/value) or documented empty baseline for P1
  5. Scripts: db:generate, db:migrate
  6. Run pnpm db:migrate against Compose Postgres

Step 6 — Studio shell & DB health

  1. Route group src/app/(studio)/page.tsx — Productizer home with Card, phase badge, Skeleton demo
  2. src/app/api/health/db/route.ts — returns JSON { ok: true } or error
  3. Client hook or server component fetch showing DB status on home
  4. Redirect / → studio or make studio the root page

Step 7 — Lib stubs (P4+ prep)

  1. pnpm add zod react-hook-form @hookform/resolvers
  2. src/lib/schemas/concept.ts — conceptFormSchema per tech-stack excerpt
  3. pnpm add @tanstack/ai (or current package name per docs)
  4. src/lib/ai/types.ts, registry.ts — interface + no-op stub provider
  5. Optional: src/components/mini-mock/MiniMock.tsx static placeholder

Step 8 — Testing & docs

  1. Vitest: pnpm add -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/jest-dom
  2. One unit test under tests/
  3. Playwright: pnpm create playwright → e2e/smoke.spec.ts
  4. Document all scripts in README; run full validation gate (see validation.html)
