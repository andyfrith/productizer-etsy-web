UX & polish (required)

Productizer is a visual creative tool—great UX and polish are not optional. Every phase should ship with:

✦Visual densityMini mocks, thumbs, skeletons
◎FeedbackLoading, empty, error states
↻Optimistic UIQuery cache + subtle motion
⌨Accessibleshadcn + focus, ARIA, keyboard
  • Design system early (P0) — Tailwind theme (Etsy-adjacent accent), typography, spacing; shadcn Button, Card, Dialog, Sheet, Toast, Skeleton.
  • Motion with purpose — Framer Motion for gallery layout, approve/reject, route transitions; avoid gratuitous animation.
  • Perceived performance — skeleton cards in concept gallery; progressive image load (thumb → full).

Local operation (Docker Compose)

Backing services run in Docker; the Next.js app runs on the host for fast HMR. Same setup for every developer.

🐳docker compose up -d
▶pnpm dev
  • docker compose up -d — Postgres (future: Redis, etc.)
  • pnpm dev — Next.js at localhost:3000
  • ./storage/ — host path for uploads & generated images (gitignored)
  • .env.local — DB URL via Compose port 5432, API keys
  • docker compose down — stop containers; named volume keeps DB

docker-compose.yml (excerpt — P0)

services:
  postgres:
    image: postgres:16-alpine
    container_name: productizer-postgres
    restart: unless-stopped
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: productizer
      POSTGRES_PASSWORD: productizer
      POSTGRES_DB: productizer
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U productizer -d productizer"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
# .env.local (host app → Compose Postgres)

STORAGE_ROOT=./storage

Core data model (excerpt)

Concept-centric schema for initial phases. Niche research tables come later.

// drizzle/schema/concepts.ts (illustrative)
export const designConcepts = pgTable('design_concepts', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: text('name').notNull(),
  description: text('description'),
  campaignLabel: text('campaign_label'), // light niche tag until research module
  styleNotes: text('style_notes'),
  status: text('status').notNull().default('draft'), // draft | active | archived
  createdAt: timestamp('created_at').defaultNow(),
});

export const conceptVariations = pgTable('concept_variations', {
  id: uuid('id').primaryKey().defaultRandom(),
  conceptId: uuid('concept_id').references(() => designConcepts.id),
  parentVariationId: uuid('parent_variation_id'),
  promptSnapshot: text('prompt_snapshot'),
  status: text('status').notNull().default('pending'), // pending | approved | rejected
  storageKey: text('storage_key').notNull(),
  thumbKey: text('thumb_key'),
  createdAt: timestamp('created_at').defaultNow(),
});

Forms & validation (excerpt)

// lib/schemas/concept.ts — shared by RHF + Server Actions
import { z } from 'zod';

export const conceptFormSchema = z.object({
  name: z.string().min(1).max(120),
  description: z.string().max(2000).optional(),
  campaignLabel: z.string().max(80).optional(),
  styleNotes: z.string().max(1000).optional(),
});

export type ConceptFormValues = z.infer<typeof conceptFormSchema>;
// components/concepts/concept-form.tsx (pattern)
const form = useForm<ConceptFormValues>({
  resolver: zodResolver(conceptFormSchema),
  defaultValues: { name: '', campaignLabel: '' },
});
