Requirements — P1 Concept CRUD

Functional and technical requirements for design concept management. Satisfies roadmap phase 1 and unblocks P2 asset upload.

Locked decisions
  • Phase: P1 — full roadmap Concept CRUD scope
  • API: REST route handlers under /api/concepts
  • Runtime: pnpm, Node 22 LTS
  • Validation bar: automated script + curl + Playwright CRUD E2E
  • Spec folder: specs/20260519-phase1-concept-crud/
  • Git branch: phase/1-concept-crud → merge via PR to master

Goal

A user can create, view, edit, and archive design concepts from the studio UI. Data persists in Postgres via Drizzle. The concept list is the primary entry point for later variation and mock workflows.

Prerequisites (P0 merged)

  • Next.js App Router, shadcn/ui, TanStack Query, Drizzle, Docker Postgres
  • src/lib/schemas/concept.ts — conceptFormSchema (extend if needed for status)
  • Studio shell at src/app/(studio)/page.tsx
  • P0 validation: ./scripts/validate-p0.sh passes on master

Requirements (MUST)

IDRequirementAcceptance
R1.1 design_concepts table Drizzle schema + migration: id (uuid), name, description, campaign_label, style_notes, status (draft | active | archived), created_at, updated_at; aligns with tech-stack excerpt
R1.2 REST list & create GET /api/concepts returns JSON array (exclude archived by default; ?includeArchived=true optional); POST /api/concepts validates body with Zod, returns 201 + concept
R1.3 REST read, update, archive GET /api/concepts/[id]; PATCH /api/concepts/[id] for editable fields; DELETE /api/concepts/[id] sets status=archived (soft delete, not hard delete)
R1.4 Concept form ConceptForm with React Hook Form + zodResolver(conceptFormSchema); shadcn Input, Textarea, Label, Button; inline field errors
R1.5 TanStack Query hooks useConcepts, useConcept(id), mutations for create/update/archive; cache invalidation on success; Sonner toast on success/error
R1.6 Concept gallery page /concepts — grid of shadcn Cards: name, campaign label, status badge, created date; Skeleton while loading; empty state CTA
R1.7 Concept detail / edit /concepts/[id] — view + edit form; archive action with confirm dialog (shadcn AlertDialog); navigate back to gallery
R1.8 Create flow /concepts/new or modal from gallery — submit creates concept and redirects to detail or gallery
R1.9 Studio navigation Studio home links to /concepts; phase badge shows P1; remove disabled “coming in P1” placeholder
R1.10 Unit tests Vitest: schema validation; API handler or service tests for create + archive (mock DB or test DB)
R1.11 E2E CRUD smoke Playwright: create concept → appears in list → edit name → archive → hidden from default list
R1.12 Validation script ./scripts/validate-p1.sh — branch, spec files, P0 regression subset, curl API, pnpm lint test build test:e2e

Requirements (SHOULD)

  • Framer Motion stagger on concept card grid (reuse P0 pattern)
  • data-testid on gallery, form, and archive confirm for E2E stability
  • Filter tabs: All / Active / Draft / Archived on gallery
  • Search by name (client-side filter acceptable for P1)
  • shadcn components: dialog, alert-dialog, badge, input, textarea, label as needed

Out of scope (P1)

  • Asset upload and concept_variations table (P2–P3)
  • Variation gallery, approve/reject, AI generation (P3–P4)
  • Poster mock, product types, export (P5–P8)
  • Concept dashboard hero thumb from assets (P6 — needs uploads)
  • Auth, multi-user, Etsy (P11+)
  • Hard delete of concepts from database

Non-functional

  • UX polish: Card layout consistent with studio shell; loading/error/empty states on all concept views
  • Security: No secrets in API responses; validate all inputs server-side with Zod
  • API errors: 400 validation, 404 not found, 500 with safe message (no stack in JSON)
  • Accessibility: Form labels, focus management in dialogs, keyboard-operable archive confirm

Dependencies

  • P0 merged to master
  • Docker Postgres running; pnpm db:migrate
  • Existing packages: zod, react-hook-form, @hookform/resolvers, @tanstack/react-query
