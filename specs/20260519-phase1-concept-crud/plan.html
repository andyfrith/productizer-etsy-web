Implementation Plan — P1 Concept CRUD

Ordered steps to implement requirements R1.1–R1.12. Each step should leave the repo in a runnable state.

Branch: phase/1-concept-crud (off master) · merge via PR only · Git workflow
Agent skills: productizer-p1-concept-crud + productizer-git-workflow + productizer-phase-validate

git fetch origin

Step 1 — Drizzle schema & migration

  1. Add designConcepts to src/lib/db/schema.ts per requirements R1.1 (uuid PK, timestamps with timezone).
  2. Extend conceptFormSchema only if needed; keep status server-controlled on create (default draft).
  3. pnpm db:generate && pnpm db:migrate
  4. Optional: seed script or dev-only POST for local testing (not required for validation).

Step 2 — Repository / service layer

  1. src/lib/concepts/repository.ts — listConcepts, getConceptById, createConcept, updateConcept, archiveConcept
  2. Use Drizzle client from src/lib/db/index.ts; map rows to API DTO (camelCase JSON).
  3. Unit-test repository or Zod parsing in isolation.

Step 3 — REST API routes

  1. src/app/api/concepts/route.ts — GET (list), POST (create)
  2. src/app/api/concepts/[id]/route.ts — GET, PATCH, DELETE (archive)
  3. Parse JSON with Zod; return consistent error shape { error: string }
  4. Verify with curl (see validation.html)

Step 4 — TanStack Query hooks

  1. src/hooks/use-concepts.ts — query keys ['concepts'], ['concepts', id]
  2. Mutations: useCreateConcept, useUpdateConcept, useArchiveConcept
  3. Invalidate list/detail on success; toast via Sonner

Step 5 — shadcn form components

  1. Add shadcn: input, textarea, label, badge, dialog, alert-dialog
  2. src/components/concepts/concept-form.tsx — controlled fields for name, description, campaignLabel, styleNotes
  3. concept-card.tsx — Card summary for gallery grid

Step 6 — Studio pages

  1. src/app/(studio)/concepts/page.tsx — gallery + empty state + link to new
  2. src/app/(studio)/concepts/new/page.tsx — create form
  3. src/app/(studio)/concepts/[id]/page.tsx — detail + edit + archive
  4. Update (studio)/page.tsx — P1 badge, active nav to concepts

Step 7 — Motion & polish

  1. Staggered card entrance on gallery (Framer Motion)
  2. Skeleton grid while useConcepts is pending
  3. Status badges: draft / active / archived color tokens

Step 8 — Tests

  1. Vitest: tests/concept-schema.test.ts — valid/invalid payloads
  2. Vitest or integration: API route tests with mocked DB if needed
  3. Playwright: e2e/concepts-crud.spec.ts — full create → edit → archive flow
  4. Add data-testid attributes per validation.html

Step 9 — Validation script & docs

  1. Ensure scripts/validate-p1.sh passes on branch
  2. README: concept routes, API summary, migrate step
  3. Run ./scripts/validate-p1.sh before PR; paste output in PR body

API contract (reference)

// POST /api/concepts
{ "name": "Sunset botanical", "description": "...", "campaignLabel": "cottagecore", "styleNotes": "..." }
→ 201 { "id": "...", "name": "...", "status": "draft", ... }

// GET /api/concepts
→ 200 [{ "id": "...", "name": "...", "status": "draft", ... }]

// PATCH /api/concepts/:id
{ "name": "Updated name" }
→ 200 { ... }

// DELETE /api/concepts/:id  (archive)
→ 200 { ..., "status": "archived" }
