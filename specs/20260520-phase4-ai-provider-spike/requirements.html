Requirements — P4 AI Provider Spike

AI-assisted 1→N variation generation with provider abstraction and streaming status. Satisfies roadmap phase 4 and unblocks P5 poster mock.

Locked decisions
  • Phase: P4 — full roadmap AI provider spike scope
  • API: REST — POST /api/concepts/:id/variations/generate (streaming response)
  • Runtime: pnpm, Node 22 LTS
  • Validation bar: automated script + curl + Playwright generate E2E (mock provider when no API key)
  • Spec folder: specs/20260520-phase4-ai-provider-spike/
  • Git branch: phase/4-ai-provider-spike → merge via PR to master

Goal

On a concept detail page, a user enters a prompt and requests 1–N AI variations. The server calls a pluggable ImageGenerationProvider (Nano Banana when configured, otherwise a mock provider for dev/CI), persists each result as a concept_variation (pending status) with prompt_snapshot, and streams job progress to the client via TanStack AI. The variation gallery refreshes when generation completes.

Prerequisites (P3 merged)

  • concept_variations + variation REST + VariationGallery approve/reject (P3)
  • prompt_snapshot column exists (nullable; populated in P4)
  • P0 stub: src/lib/ai/types.ts, registry.ts, @tanstack/ai in package.json
  • P3 validation: ./scripts/validate-p3.sh passes on master

Requirements (MUST)

IDRequirementAcceptance
R4.1 Provider types Extend GenerateImageInput: prompt, optional referenceImagePath or buffer from sourceAssetId, width/height; GenerateImageResult returns image bytes or temp path before persistence
R4.2 Nano Banana provider src/lib/ai/providers/nanobanana.ts implements ImageGenerationProvider; reads NANOBANANA_API_KEY; calls vendor HTTP API (document endpoint in code comment); throws clear error when key missing and mock not forced
R4.3 Mock provider for CI src/lib/ai/providers/mock.ts copies e2e/fixtures/sample.png (or solid-color sharp buffer) per variation; used when NANOBANANA_API_KEY empty or AI_PROVIDER=mock
R4.4 Provider registry getImageProvider(id?) resolves AI_PROVIDER from env (nanobanana | mock); auto-fallback to mock when key absent; Vitest covers resolution
R4.5 Generate request schema Zod: prompt (min 3, max 2000), count (int 1–4, default 1), optional parentVariationId, optional sourceAssetId, optional providerId
R4.6 Generate REST route POST /api/concepts/:id/variations/generate validates body + concept exists; 404 if concept missing; 400 on invalid asset/parent refs
R4.7 Persist generated variations For each of count images: create variation via existing repository (sharp renditions, storage keys); status=pending; set prompt_snapshot to submitted prompt; optional parent_variation_id / source_asset_id
R4.8 Streaming progress events Route returns text/event-stream (or NDJSON) events: queued, generating (index, total), saved (variationId), done (variationIds[]), error (message). Client parses with TanStack AI or fetch reader
R4.9 useGenerateVariations src/hooks/use-generate-variations.ts — exposes generate, status, progress, error; on done invalidates ['variations', conceptId] and concept detail
R4.10 Generate UI panel On concept detail above gallery: prompt textarea, count select (1–4), Generate button data-testid="variation-generate-submit"; progress text data-testid="variation-generate-status"; disabled while in-flight; Sonner on error
R4.11 Reference image for generate When concept has previewAssetId, optional “Use preview as reference” sends sourceAssetId; provider receives reference image bytes when supported
R4.12 Unit tests Vitest: generate Zod schema; registry mock fallback; optional nanobanana request builder (mocked fetch)
R4.13 E2E generate flow Playwright e2e/variation-generate.spec.ts: with mock provider, submit prompt count=2, wait for status done, assert 2 new pending cards in variation-gallery
R4.14 Validation script ./scripts/validate-p4.sh with --tier docs before implementation; full tier before PR
R4.15 Env documentation .env.example documents AI_PROVIDER, NANOBANANA_API_KEY; README section “AI generation (P4)”

Out of scope

  • Poster / product mock compositing (P5)
  • Studio dashboard hero + MiniMock home (P6)
  • Persistent generation_jobs table + Redis/BullMQ queue (defer; in-process stream suffices for P4 spike)
  • Auto-approve AI variations (user still approves in gallery)
  • Auth, billing, rate limits per user
  • Additional providers beyond Nano Banana + mock (registry extensible only)
  • Etsy, niche research
